# Dashboard 前端测试用例（手工 + 轮询校验）

适用页面：`/docs/fars_dashboard.html`（由 Flask 静态服务直接打开 `http://127.0.0.1:8080/`）

## 0. 测试前置

- 服务已启动：`python server.py`
- 浏览器无代理干扰（或能正常访问 LLM provider 的 `base_url`）
- 建议先执行一次「从0开始」（保留下载中心与 archives）以保证状态干净

## 1. LLM 配置（配置面板）

### TC-LLM-01 读取生效配置

- 步骤
  - 打开 Dashboard
  - 观察 LLM 配置区域（provider/model/base_url/max_tokens 等）
- 期望
  - 展示内容与后端 `GET /api/config/llm` 一致
  - `api_key` 不回显明文（仅显示“已配置/未配置”之类的状态）

### TC-LLM-02 保存配置并生效（local 覆盖）

- 步骤
  - 修改 provider/model/base_url/max_tokens/temperature
  - 点击保存
  - 刷新页面
- 期望
  - 保存成功提示
  - 刷新后仍保持新配置
  - 后端 `config.local.json` 覆盖生效（不影响 `config.json`）

### TC-LLM-03 LLM 未就绪时按钮禁用

- 步骤
  - 将 provider 设置为 `openai`，但 api_key 为空
  - 保存并刷新
- 期望
  - 「开始/继续/下一篇」按钮禁用
  - 状态区提示 LLM 未配置原因（provider、model、base_url 可见，key 不泄露）

## 2. 控制按钮（开始/暂停/继续/停止/从0开始）

### TC-CTRL-01 从0开始（不删除 archives 与下载中心）

- 步骤
  - 点击「从0开始」
  - 刷新页面
- 期望
  - 研究状态恢复到初始（papers/hypotheses/experiments 清空或回到初始卡片）
  - 下载中心仍可下载历史种子文献（seed papers）
  - 后端返回 `backup_dir` 且 `remove_archives=false`

### TC-CTRL-02 开始生成（启动研究流水线）

- 步骤
  - 点击「开始」
  - 观察状态区与实验记录区变化（每 2 秒自动刷新）
- 期望
  - `research_status` 从 idle → starting → hypothesis/experimenting/writing 等推进
  - 实验记录卡片 `exp_1/exp_2/exp_3` 状态随阶段变化

### TC-CTRL-03 暂停 / 继续

- 步骤
  - 生成进行中点击「暂停（下一篇后暂停）」或「暂停」
  - 等待状态变为 paused（或完成当前一篇后 paused）
  - 点击「继续」
- 期望
  - 按钮显示状态正确切换（暂停时只显示继续/停止）
  - `is_paused/is_generating` 与按钮状态一致

### TC-CTRL-04 停止

- 步骤
  - 生成进行中点击「停止」
  - 观察状态区与按钮状态
- 期望
  - 生成停止，状态区提示已停止
  - 「开始」恢复可用，「暂停/继续/停止」隐藏或禁用

## 3. 实验记录（研究流程阶段三卡片）

说明：实验记录代表“研究流程阶段”，不是模型参数对比实验。

### TC-EXP-01 初始展示（3 张卡片 + pending）

- 前置
  - 执行过「从0开始」
- 步骤
  - 打开 Dashboard
- 期望
  - 固定 3 张卡片：
    - 实验1：文献综述与假设生成
    - 实验2：引用/作者图谱构建
    - 实验3：写作与产物落盘
  - 未开始阶段显示 `… 等待`（pending）
  - 时间区间显示 `未知 - 未开始` 或同等语义

### TC-EXP-02 阶段推进与时间戳

- 步骤
  - 点击「开始」
  - 观察三个卡片的 `status / started_at / completed_at` 变化
- 期望
  - 进行中阶段显示 `◐ 进行中`，完成阶段显示 `✓ 成功`
  - 每个阶段被激活时 `started_at` 写入；完成时 `completed_at` 写入

### TC-EXP-03 指标字段（进度/成本/产物）

- 步骤
  - 生成进行中，观察每张卡片 metrics tag
- 期望
  - 每张卡都有：
    - `进度%`
    - `LLM调用`
    - `Token`
    - `失败`
  - 阶段 1 额外包含：
    - `种子论文`
    - `假设数`
  - 阶段 3 额外包含：
    - `产物论文`

### TC-EXP-04 LLM 超时降载可用性（避免 504 直接失败）

- 步骤
  - 设置较大的 `max_tokens`（例如 32000）并开始生成
  - 若上游偶发 504，继续观察
- 期望
  - 页面不应因为一次 504 立即崩溃
  - 后端会自动降低单次输出 token 并继续分段续写（最终尽可能完成）
  - 若仍失败，失败原因应清晰可读（而不是 JSONDecodeError）

## 4. 拓扑图（论文引用/关系图）

### TC-GRAPH-01 节点显示短标题

- 步骤
  - 打开拓扑图区域
  - 观察节点文字
- 期望
  - 节点直接显示论文标题的短截断文本（不是 RS-xxx）
  - hover/点击可看到全称与 research_id（若当前实现为 tooltip/modal）

## 5. 作者-机构网络图

### TC-AUTH-01 节点类型与边

- 步骤
  - 打开作者关系图区域
  - 等待数据加载
- 期望
  - 至少包含：
    - 作者节点
    - 机构节点
    - 作者-作者合作边（权重随共同论文数增加）
    - 作者-机构隶属边（若能抽取到 affiliation）

### TC-AUTH-02 合作边权重渲染

- 步骤
  - 观察作者-作者的连线粗细/强度
- 期望
  - 合作次数越多的边越“粗/亮”（权重可视化生效）

## 6. 下载中心

### TC-DL-01 种子文献可下载

- 步骤
  - 打开下载中心
  - 下载任意种子论文 PDF
- 期望
  - 下载成功
  - 重新开始（从0开始）后仍可下载（不被清理）

### TC-DL-02 研究产物可下载

- 前置
  - 至少成功生成 1 篇论文
- 步骤
  - 下载论文 markdown / 指标数据 / 回测数据
- 期望
  - 下载链接有效（200）
  - 文件内容与研究目录一致
